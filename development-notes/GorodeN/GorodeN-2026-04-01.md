## День 30 (01.04.2026)

Занимался ревью PR ребят.

Поставил Issue GitHub и тикет в Trello на написание тестов. Написал тесты для MobX стора авторизации (AuthStore) и для серверных эндпоинтов.

**Клиентские тесты AuthStore**:
- Создал файл `src/store/__tests__/AuthStore.test.ts`
- Написал 10 тестовых кейсов, покрывающих:
  - геттер `isAuthenticated`
  - восстановление сессии (`restoreSession`) - успех, 401, сетевая ошибка
  - вход (`login`) - успех, ошибка
  - регистрация (`register`) - успех, ошибка
  - выход (`logout`) - успех, ошибка (состояние очищается в любом случае)
- Использовал моки для `authService`, `notify` и `setAccessToken`
- Возникла проблема с моком `services/api`, пришлось поковыряться с алиасами в `jest.config.cjs`, поспрашивать у ИИ

**Серверные тесты**:
- Установил зависимости по рекомендациям из интернета и ИИ: `mongodb-memory-server`, `supertest`, `babel-jest`, `jest`
- Настроил Jest для сервера
- В `server/src/index.js` вынес `export const app` и обернул запуск сервера в условие `if (process.env.NODE_ENV !== 'test')`.
- Написал интеграционные тесты для всех эндпоинтов авторизации:
  - `/register` - проверка создания пользователя, дубликата email
  - `/login` - успешный вход, неверные данные
  - `/refresh` - обновление access токена, отсутствие токена
  - `/logout` - удаление токена и очистка cookie
  - `/profile` - доступ с токеном, отсутствие токена
- Столкнулся с ошибкой, что корневой Jest пытался обработать серверные файлы и падал на `import`. Решил, добавив `testPathIgnorePatterns: ['/node_modules/', '/server/']` в корневой `jest.config.cjs`. Также добавил скрипт `test:server` в корневой `package.json` для удобного запуска серверных тестов.
